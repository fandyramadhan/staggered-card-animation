Create a responsive landing page in Next.js for a mental-wellness brand ("CODESIGN") with a scroll-driven, pinned, staggered card reveal. As the user scrolls into the features section, the section pins, the heading slides up, and three colored cards fly in from the top and bottom in a staggered sequence until they settle into a row. Use GSAP + ScrollTrigger (loaded via CDN).

CORE IDEA:
- Page has three parts: a HERO (purple rounded panel), a FEATURES section (the pinned staggered cards), and a big black AWWWARDS-style FOOTER.
- HERO: an SVG background shape "draws" itself with a path-trim animation (1.5s); when it finishes, the headline / subtitle / button slide up from below.
- FEATURES: the section is PINNED for one viewport. Initial state shows only the centered heading + subtitle. On scroll, the heading travels to the top while three cards slide in — two from above, one from below — STAGGERED (0.15 between each) — landing into a 3-column row.
- Cards do a subtle 3D tilt that follows the mouse on hover.
- All scroll animation is tied to scroll progress (scrub), so it scrubs forward/backward as the user scrolls.

CONSTANTS:
const COLORS = { hero: "#573FEF", pink: "#FF29AE", blue: "#2E74FF", green: "#7AC743", ink: "#131313" }
const PIN_SCROLL = "+=2200"   // extra scroll length the pinned timeline consumes
const STAGGER = 0.15          // delay between each card in the timeline
const CARD_OFFSET = 1.15      // multiple of viewport height each card starts off-screen
const DRAW_DURATION = 1.5     // s, hero shape path-trim
const TILT_MAX = 10           // deg, max 3D tilt on hover
const EASE = "power3.out"
// Cards cycle in fixed order; `from` decides entry direction.

GSAP LOADER (client hook — GSAP via CDN, in order):
"use client"
import { useEffect, useState } from "react"
// Inject gsap.min.js then ScrollTrigger.min.js (order matters), register the
// plugin once both load, and return `ready`. Resolve nothing / leave ready=false
// if the CDN fails so a fallback can reveal content.
const SCRIPTS = [
  "https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.5/gsap.min.js",
  "https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.5/ScrollTrigger.min.js",
]
export default function useGsap() {
  const [ready, setReady] = useState(false)
  useEffect(() => {
    let cancelled = false
    ;(async () => {
      for (const src of SCRIPTS) await loadScript(src)   // append <script>, await onload
      if (cancelled) return
      if (window.gsap && window.ScrollTrigger) window.gsap.registerPlugin(window.ScrollTrigger)
      setReady(true)
    })()
    return () => { cancelled = true }
  }, [])
  return ready
}

CORE COMPONENT (the staggered cards — client component):
"use client"
import { useEffect, useRef } from "react"
import useGsap from "./useGsap"

export default function Features() {
  const ready = useGsap()
  const sectionRef = useRef(null)      // ScrollTrigger trigger
  const pinRef = useRef(null)          // the element that gets pinned (one viewport tall)
  const titleGroupRef = useRef(null)   // heading + subtitle wrapper (travels center -> top)
  const revealRefs = useRef([])        // inner text spans (slide-up reveal on enter)
  const cardsRef = useRef([])          // the three cards
  revealRefs.current = []
  cardsRef.current = []
  const addReveal = (el) => el && !revealRefs.current.includes(el) && revealRefs.current.push(el)
  const addCard   = (el) => el && !cardsRef.current.includes(el)   && cardsRef.current.push(el)

  // --- scroll animations ---
  useEffect(() => {
    if (!ready) return
    const gsap = window.gsap
    const section = sectionRef.current, pin = pinRef.current, titleGroup = titleGroupRef.current
    const cards = cardsRef.current

    const ctx = gsap.context(() => {
      // 1) Heading + subtitle slide up from below when the section first enters.
      gsap.fromTo(revealRefs.current,
        { yPercent: 70, opacity: 0 },
        { yPercent: 0, opacity: 1, duration: 0.9, ease: EASE, stagger: 0.12,
          scrollTrigger: { trigger: section, start: "top 70%", once: true } })

      const mm = gsap.matchMedia()

      // 2) Desktop / tablet: PIN + scrubbed staggered reveal.
      mm.add("(min-width: 768px)", () => {
        gsap.set(cards, { opacity: 1 })
        const tl = gsap.timeline({
          scrollTrigger: {
            trigger: section, start: "top top", end: PIN_SCROLL,
            scrub: 1, pin: pin, anticipatePin: 1, invalidateOnRefresh: true,
          },
        })
        // heading travels from vertical center of the viewport up to its resting top
        // position. offsetTop/offsetHeight are layout-based, unaffected by transforms.
        tl.fromTo(titleGroup,
          { y: () => pin.clientHeight / 2 - (titleGroup.offsetTop + titleGroup.offsetHeight / 2) },
          { y: 0, ease: "none" }, 0)
        // cards slide in — top cards from above, bottom card from below — staggered.
        tl.fromTo(cards,
          { y: (i) => (CARDS[i].from === "bottom" ? 1 : -1) * (window.innerHeight * CARD_OFFSET) },
          { y: 0, ease: EASE, duration: 1, stagger: STAGGER }, 0)
      })

      // 3) Mobile: no pin — simple staggered fade/slide-up reveal.
      mm.add("(max-width: 767px)", () => {
        gsap.fromTo(cards,
          { y: 60, opacity: 0 },
          { y: 0, opacity: 1, duration: 1, ease: EASE, stagger: STAGGER,
            scrollTrigger: { trigger: pin, start: "top 75%" } })
      })
    }, sectionRef)

    return () => ctx.revert()
  }, [ready])

  // --- 3D tilt following the mouse (max TILT_MAX deg) ---
  useEffect(() => {
    if (!ready) return
    const gsap = window.gsap
    const cleanups = []
    cardsRef.current.forEach((card) => {
      const onMove = (e) => {
        const r = card.getBoundingClientRect()
        const px = (e.clientX - r.left) / r.width, py = (e.clientY - r.top) / r.height
        gsap.to(card, { rotateY: (px - 0.5) * TILT_MAX * 2, rotateX: (0.5 - py) * TILT_MAX * 2,
          duration: 0.4, ease: EASE, transformPerspective: 800, transformOrigin: "center" })
      }
      const onLeave = () => gsap.to(card, { rotateX: 0, rotateY: 0, duration: 0.6, ease: EASE })
      card.addEventListener("mousemove", onMove)
      card.addEventListener("mouseleave", onLeave)
      cleanups.push(() => { card.removeEventListener("mousemove", onMove); card.removeEventListener("mouseleave", onLeave) })
    })
    return () => cleanups.forEach((fn) => fn())
  }, [ready])

  return (
    <section id="features" ref={sectionRef} className="relative">
      <div ref={pinRef} className="relative flex min-h-screen flex-col items-center overflow-hidden px-4 pt-[8vh] sm:px-6">
        <div ref={titleGroupRef} className="text-center">
          <h2 className="text-[34px] font-extrabold leading-[1.1] tracking-tight text-ink sm:text-5xl md:text-[52px]">
            <span ref={addReveal} className="reveal-text">Everything you need,</span><br />
            <span ref={addReveal} className="reveal-text">in one place.</span>
          </h2>
          <p className="mt-5 text-sm text-neutral-500 sm:text-base">
            <span ref={addReveal} className="reveal-text">Simple tools to help you feel better, one day at a time.</span>
          </p>
        </div>
        <div className="mt-[5vh] grid w-full max-w-[1240px] grid-cols-1 gap-6 md:grid-cols-3">
          {CARDS.map((card) => (
            <article key={card.title.join(" ")} ref={addCard}
              className={`feature-card flex flex-col rounded-[28px] ${card.bg} p-6 sm:p-7`}>
              <img src={card.img} alt={card.title.join(" ")} className="h-auto w-full" draggable="false" />
              <h3 className="mt-6 text-[24px] font-extrabold leading-[1.12] tracking-tight text-white sm:text-[28px]">
                {card.title[0]}<br />{card.title[1]}
              </h3>
            </article>
          ))}
        </div>
      </div>
    </section>
  )
}

HERO (path-trim shape, then text slide-up):
- Inline the SVG shape so its <path> can be animated. Measure length with path.getTotalLength().
- gsap.set(path, { strokeDasharray: length, strokeDashoffset: length }) then a timeline:
    tl.to(path, { strokeDashoffset: 0, duration: DRAW_DURATION, ease: EASE })
      .to(textItems, { opacity: 1, yPercent: 0, duration: 0.9, ease: EASE, stagger: 0.12 }, ">-0.1")
- The hero panel is a rounded box on COLORS.hero; the SVG sits absolute inset-0 (preserveAspectRatio="xMidYMid slice"), content sits relative z-10 above it.

FOOTER (awwwards-style):
- Full-width black (COLORS.ink) section; giant wordmark "CODESIGN" with text-[clamp(3rem,19vw,22rem)] font-extrabold leading-[0.8] so it fills the viewport width.
- Above it: tagline + CTA button + link columns; below it: copyright bar.

STYLING / TAILWIND:
- Font: Inter via next/font — extrabold (800) for headings/titles, regular (400) for body.
- Tailwind theme colors: hero, card-pink, card-blue, card-green, ink (from COLORS).
- Cards: rounded-[28px], colored bg, white text. Illustrations are transparent .webp — render them with NO background wrapper so they sit directly on the card color.
- Pre-animation CSS to avoid a flash before GSAP runs:
    .feature-card { opacity: 0; transform-style: preserve-3d; will-change: transform, opacity }
    .reveal-text  { opacity: 0; display: inline-block; will-change: transform, opacity }

KEY POINTS:
- "Staggered" comes from the single `stagger: STAGGER` on the cards tween — cards start moving 0.15 apart, so pink → blue → green, never all at once.
- The whole card move is on a `scrub` timeline, so the stagger is spread across scroll progress (scrubs both ways) instead of playing on a fixed timer.
- Heading travel uses offsetTop/offsetHeight (layout values), NOT getBoundingClientRect, so an active transform doesn't skew the centering math. Pair with invalidateOnRefresh so it recomputes on resize.
- Cards start ±CARD_OFFSET * viewport height off-screen; overflow-hidden on the pinned box clips them so the initial state looks empty (just the heading).
- Wrap GSAP work in gsap.context(...) scoped to the section and return ctx.revert() for clean unmount.
- Use gsap.matchMedia so desktop pins/scrubs and mobile falls back to a plain staggered reveal (pinning + horizontal rows are bad on small screens).
- 3D tilt animates rotateX/rotateY (max TILT_MAX) with transformPerspective; it only touches rotation, so it never fights the timeline's `y`.

NOTES:
- This is scroll-driven; verify by scrolling, not by a fixed timer. Tune feel via PIN_SCROLL (scroll length), STAGGER, and CARD_OFFSET.
- Loads GSAP from CDN (per requirement) — add a setTimeout fallback that force-reveals text/cards if the CDN never loads, so nothing stays hidden.
- Never run `next build` while `next dev` is running — they share .next and the dev webpack cache gets corrupted.

TECH: Next.js (App Router), JavaScript/JSX, Tailwind CSS, GSAP 3 + ScrollTrigger (via CDN), Inter (next/font).
DO NOT measure the heading's centering offset with getBoundingClientRect — an active GSAP transform skews it; use layout-based offsetTop/offsetHeight instead.
